一、项目简介

Quant 是一个面向普通投资者和量化研究初学者的策略生成项目。项目的目标，是让用户通过自然语言直接描述交易想法，由系统自动解析策略意图、匹配模板，并生成可在聚宽平台回测的 Python 策略代码。

⭐ 本项目已实现从自然语言到量化策略代码的完整自动化生成流程，并具备网页工具与可执行分发版本。

系统整体形成了如下闭环：

自然语言输入 → AI解析 → 结构化参数 → 模板生成 → JoinQuant策略代码输出

该项目不仅是一个代码生成工具，更接近一个“量化策略编译器”。

二、项目背景与开发历程

项目从一个最小可运行原型（MVP）开始，仅支持简单动量策略的生成。随着开发推进，逐步围绕真实使用场景进行了系统性扩展，核心演进包括：

从单一策略扩展为多策略模板系统
从命令行工具升级为本地网页应用（Streamlit）
解决 JoinQuant 代码兼容问题，统一输出规范
增加 Windows 可执行程序，实现脱离开发环境运行
优化项目结构并整理上传 GitHub

在持续迭代过程中，项目逐渐从“脚本工具”演变为具备工程结构的系统。

三、项目核心功能
支持中文自然语言输入策略描述
基于大模型（DeepSeek）自动解析策略类型与参数
支持多种策略模板自动生成 JoinQuant 回测代码
提供本地网页界面（输入、生成、展示一体化）
自动保存生成结果到 generated_strategy.py
支持打包为 Windows 可执行程序（.exe）进行分发
四、系统工作流程

当前系统完整流程如下：

用户在网页中输入策略描述
（如：“做一个沪深300的KDJ策略，最多持有10只股票”）

前端调用核心函数：

generate_and_save_strategy()
agent/ai_parser.py
将自然语言解析为统一 JSON 参数
agent/code_generator.py
根据 strategy_type 调用对应策略模板
templates/
生成符合 JoinQuant 规范的策略代码
输出：
写入 generated_strategy.py
同时展示在网页界面
五、当前支持的策略模板

当前项目支持以下策略类型：

1. 动量策略（momentum）

基于历史收益率排序，选择表现最强的股票进行持仓。

👉 已优化：

支持统一股票池（全市场 / HS300 / ZZ500）
修复股票池为空问题
2. 均线突破策略（ma_breakout）

基于价格与均线关系判断买卖信号，适用于单标的交易。

3. KDJ 择时策略（kdj_timing）

基于 KDJ 指标进行择时或选股。

👉 已优化：

参数结构标准化
阈值逻辑增强
支持多股票池
4. 羊驼轮换策略（alpaca_rotation）

定期卖出表现最差资产并随机补入新标的。

👉 已优化：

调仓逻辑更清晰
持仓控制更稳定
5. Brandes 价值策略（brandes_value）

基于估值、ROE、现金流等财务指标进行选股。
